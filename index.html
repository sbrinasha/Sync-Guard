SyncGuard

AI-Driven Smart Home Automation System for Adaptive Safety

Bridging the gap between physical hardware and intelligent software to eliminate "alarm fatigue" using sensor fusion and machine learning.

Explore the Code

The Problem: Alarm Fatigue

Traditional smart home safety systems rely on fixed-threshold logic, triggering immediate alarms upon detecting environmental changes. This lack of contextual verification frequently leads to false positives—such as a spilled drink triggering a rain protocol or transient humidity spikes closing windows unnecessarily. Over time, this diminishes user trust.

SyncGuard solves this. By cross-referencing local sensor data with global meteorological forecasts, it implements an Adaptive Safety framework that intelligently distinguishes between localized anomalies and genuine weather threats.

System Architecture

📟

Edge Layer

Powered by an ESP32-S3 microcontroller, gathering micro-climate telemetry using a BME280 sensor and analog rain sensors.

📡

Communication

Low-latency telemetry streaming using the lightweight MQTT protocol (Eclipse Mosquitto broker) via JSON payloads.

🧠

AI Engine

A Python-based backend that fuses local data with OpenWeather API forecasts, analyzed by a Random Forest Classifier.

💻

Presentation

A live Flask-based dashboard providing real-time telemetry visualization, AI state indication, and manual override controls.
